# Exercise 3 - Sorting Customer Orders

## Objective

The objective of this exercise is to implement sorting algorithms to sort customer orders based on their total price and compare their performance.

## Description

In an e-commerce platform, customer orders need to be sorted according to their total price. This helps in identifying and prioritizing high-value orders. In this exercise, Bubble Sort and Quick Sort algorithms are implemented to sort customer orders.

## Order Details

Each order contains the following attributes:

* Order ID
* Customer Name
* Total Price

## Bubble Sort

Bubble Sort repeatedly compares adjacent elements and swaps them if they are in the wrong order.

### Time Complexity

* Best Case: O(n)
* Average Case: O(n²)
* Worst Case: O(n²)

## Quick Sort

Quick Sort selects a pivot element and partitions the array around the pivot. It is one of the most efficient sorting algorithms.

### Time Complexity

* Best Case: O(n log n)
* Average Case: O(n log n)
* Worst Case: O(n²)

## Comparison

Quick Sort performs much faster than Bubble Sort for large datasets because it reduces the number of comparisons and swaps.

## Result

The customer orders were successfully sorted using Bubble Sort and Quick Sort based on their total price. The sorted output was displayed correctly.
